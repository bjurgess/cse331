Project 4 FAQ

Question:

I was wondering if we were allowed to change the class Node.

Answer:

If you take a closer look at the project description, in the last paragraph I say: "As always you may not change the signatures of the public methods, the node class, or the way in which the tree class interacts with the rest of the program." So the answer is no, you may not modify the node class. If you did change the node class you would run the risk of breaking another part of the code that is not a part of the project.

Question:

I see that there is a m_active for each node, is this so that we may use lazy deletion?

Answer:

You must actually delete the nodes from memory and update the tree.

Question:

How should we handle inserting a node that is already in the tree.

Answer:

Tree's only hold one copy of each item.

Question:

Can we add a pointer to the parent node in the node class?

Answer:

No, but you can implement functions in the private space, and you can pass those functions pointers, and pointers by reference. That would be a very elegant way to implement delete.

Question:

For number 1 on the written portion what values should we use for the heights?

Answer:

Once you've compiled your completed project, load the entire data set and use the values it gives you for the height.
